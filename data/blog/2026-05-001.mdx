이 글은 `PhotoPost` 컴포넌트의 동작을 시각적으로 확인하기 위한 테스트 페이지입니다.
각 섹션은 서로 다른 상황(사진 여러 장, 한 장, 글이 긴 경우, 짧은 경우)을 보여줍니다.

## 1. 여러 장 + 적당한 길이의 글

가장 일반적인 사용 케이스입니다. 사진 위에 커서를 올리면 양 끝에 반투명 화살표가 나타나며,
화살표를 눌러 다음/이전 사진으로 넘어갈 수 있습니다.

<PhotoPost photos={[
  { src: '/static/images/canada/lake.jpg', alt: '캐나다의 한 호수' },
  { src: '/static/images/canada/maple.jpg', alt: '단풍나무' },
  { src: '/static/images/canada/mountains.jpg', alt: '산맥' },
  { src: '/static/images/canada/toronto.jpg', alt: '토론토 도심' }
]}>

### 캐나다에서

첫 번째 사진은 호수입니다. 잔잔한 수면 위로 비치는 구름이 인상적이었습니다.

두 번째는 단풍. 가을이라 곳곳에 붉은 잎이 가득했고, 길을 걷다 잠시 멈춰 사진을 찍지 않을 수 없었습니다.

세 번째는 산맥. 멀리 보이는 능선이 끝없이 이어져 있었습니다.

네 번째는 토론토 시내. 자연 풍경과는 다른 결의 매력이 있었습니다.

</PhotoPost>

## 2. 한 장 + 매우 긴 글 (스크롤 동작 확인)

사진이 한 장이면 화살표가 표시되지 않습니다. 글이 정사각형 영역보다 길어지면 우측에 스크롤바가 생깁니다.

<PhotoPost photos={['/static/images/canada/mountains.jpg']}>

### 산을 오르며

처음에는 가벼운 마음으로 출발했습니다. 길이 평탄해 보였고, 날씨도 좋았기 때문입니다.

하지만 한 시간쯤 걷자 경사가 가팔라지기 시작했습니다. 숨이 차고 다리가 무거워졌습니다.

중간 지점에서 잠시 쉬며 물을 마셨습니다. 멀리 보이는 풍경이 점점 더 넓어지고 있었습니다.

다시 출발. 이번에는 좀 더 천천히, 호흡에 집중하며 걸었습니다.

정상에 가까워질수록 바람이 강해졌고, 기온도 떨어졌습니다. 준비해 온 외투를 꺼내 입었습니다.

마지막 구간은 거의 기다시피 올랐습니다. 손과 발을 모두 써야 했고, 몇 번이고 멈춰 숨을 골라야 했습니다.

정상에 도착한 순간, 모든 피로가 사라지는 것 같았습니다. 사방으로 펼쳐진 산맥과 그 너머의 호수가 한눈에 들어왔습니다.

한참을 그 자리에 앉아 있었습니다. 바람 소리 외에는 아무 소리도 들리지 않았습니다.

내려오는 길은 올라갈 때보다 짧게 느껴졌습니다. 같은 풍경인데도 다른 느낌이었습니다.

</PhotoPost>

## 3. 한 장 + 짧은 글

글이 정사각형보다 짧으면 스크롤은 생기지 않고, 글 영역 윗부분에 자연스럽게 배치됩니다.

<PhotoPost photos={['/static/images/canada/maple.jpg']}>

가을. 단 한 단어로 충분한 풍경.

</PhotoPost>

## 4. 가로형/세로형 사진 (letterbox 확인)

원본 비율이 정사각형이 아니어도 비율을 유지한 채 정사각형 안에 맞춰집니다. 남는 공간은 배경색으로 채워집니다.

<PhotoPost photos={[
  { src: '/static/images/canada/toronto.jpg', alt: '토론토 (가로 사진)' },
  { src: '/static/images/canada/lake.jpg', alt: '호수' }
]}>

가로가 긴 사진은 위아래 여백이, 세로가 긴 사진은 좌우 여백이 생깁니다.
배경색은 라이트 모드에서 `gray-50`, 다크 모드에서 `gray-900`을 사용합니다.

화살표로 두 사진을 번갈아 보면 비율 차이가 확인됩니다.

</PhotoPost>
